4. 提供 Vercel 部署指引

### 选项 2：手动操作（完全控制）

#### 步骤 1：创建 GitHub 仓库

1. 访问 https://github.com/new
2. Repository name: `short-video`（或你喜欢的名字）
3. 选择 **Public**
4. **不要**勾选任何初始化选项
5. 点击 **Create repository**

#### 步骤 2：推送代码到 GitHub

在终端中运行（替换 `YOUR_USERNAME` 为你的 GitHub 用户名）：

```bash
cd /Users/lihenry/Documents/short_video

# 添加远程仓库
git remote add origin https://github.com/YOUR_USERNAME/short-video.git

# 推送代码
git push -u origin main
```

**注意**：如果需要认证，使用 Personal Access Token：
- 获取地址：https://github.com/settings/tokens
- 权限：选择 `repo`
- 使用 Token 作为密码

#### 步骤 3：部署到 Vercel

##### 方式 A：网页部署（推荐）

1. 访问 https://vercel.com
2. 点击 **Sign Up** 或 **Log In**
3. 选择 **Continue with GitHub**
4. 授权 Vercel 访问你的 GitHub
5. 点击 **Add New...** → **Project**
6. 从列表中找到 `short-video` 仓库
7. 点击 **Import**
8. 保持默认配置，点击 **Deploy**
9. 等待 1-2 分钟，部署完成！
10. 你会看到：
    - ✅ 部署成功的庆祝动画
    - 🔗 访问链接（如：`https://short-video-xxx.vercel.app`）
    - 📱 可以直接点击访问或分享

##### 方式 B：命令行部署

```bash
# 安装 Vercel CLI（只需一次）
npm install -g vercel

# 登录 Vercel
vercel login

# 部署
cd /Users/lihenry/Documents/short_video

# 按照提示操作：
# - Setup and deploy? Y
# - Which scope? 选择你的账号
# - Link to existing project? N
# - Project name? 按回车使用默认
# - In which directory? 按回车使用当前目录
# - Override settings? N
```

## ⚠️ 常见问题

### Q1: GitHub 推送失败？

**原因**：需要认证

**解决**：
1. 访问 https://github.com/settings/tokens
2. 点击 **Generate new token** → **Generate new token (classic)**
3. 选择 `repo` 权限
4. 生成并复制 Token
5. 推送时使用 Token 作为密码

### Q2: Vercel 找不到我的仓库？

**解决**：
1. 在 Vercel 设置中检查 GitHub 集成
2. 确保仓库是 Public
3. 重新授权 Vercel 访问 GitHub

### Q3: 视频无法播放？

**可能原因**：
- 视频格式不兼容（需要 MP4 + H.264 编码）
- 移动浏览器限制自动播放
- 网络问题

**解决**：
- 检查视频格式
- 用户需要点击屏幕开始播放
- 检查网络连接

### Q4: 部署后页面空白？

**检查**：
1. 浏览器控制台是否有错误
2. Vercel 部署日志是否成功
3. 视频路径是否正确
